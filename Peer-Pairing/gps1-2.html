MJ Schindler and Caroline Artz

What is Pair Programming?

According to Wikipedia, pair programming is an agile software development technique in which two programmers work as a pair together on one workstation. One, the driver, writes code while the other, the navigator, reviews each line of code as it is typed in. The two programmers switch roles frequently.

Why is Pair Programming Important to Dev Bootcamp?

Dev Bootcamp uses pair programming to train us in a best practice that we may see on the job and to help us learn from each other.

Driver VS. Navigator

Driver

  • Focus on the immediate problem at hand
  • Don't type passively
  • Say what you are about to do
  • Ask for an implementation idea
  • Ask for a better way to solve the problem at hand

Navigator

  • Focus on the big picture
  • Don't dictate the code
  • Bring up alternative ideas
  • Point out possible inputs that the code doesn't cover
  • Suggest clearer names for variables and subroutines
