Absolute, Relative, or Fixed Positioning: What's the difference?

When to Stay Static and When to Re-Position

February 15, 2015

During this second week of Dev Bootcamp, we've brushed up on our HTML and CSS skills. One aspect of CSS that I find most challenging is the positioning property.

In CSS, the position property tells elements on a webpage how to position themselves. There are four different values that the position property may have (well, there are five, counting the inherit value, but we'll ignore that one for now):

Static

position: static;

"Static" is the default position for elements on your page. If you don't change the position in CSS, elements will arrange themselves according to the order in which they are mentioned in the HTML. This is called "normal flow." The elements will display one at a time, from left to right, top to bottom.

The static position is the only one that you can't adjust by adding the "top," "right," "bottom," and "left" properties. That means that you would never create a class with these properties:

.static-position { position: static; top: 5px; right: 5px; }

Static position sounds simple, right? Let's get a bit more complicated.

Relative

position: relative;

When you set an element to have a relative position, you're telling it, "Position yourself in relation to where you would have been if you were still static, in normal flow." When you alter its position by using the "top," "right," "bottom," and "left" properties, you're saying, "Go X pixels this way from where you used to be.

Absolute

position: absolute;

The absolute position is like a rebellious kid that have to keep in line with really, REALLY specific orders. When you set an element to an absolution position, you've taken it completely out of normal flow. You've told it, "Hey element, you ABSOLUTELY must position yourself X pixels this way and Y pixels in relation to your parent element. And if you don't have a parent element with a position other than static, position yourself X pixels this way and Y pixels in relation to the screen." Now that you've put that kid in his place using the "top," "right," "bottom," and "left" properties, all the other elements will ignore it. They will not position themselves in relation to each other at all.

Fixed

position: static;

The fixed position is similar to the absolute position. When you use it on an element, you take it out of normal flow and tell it stay in a specific location on your screen. Unlike an element in the absolute position, an element in the fixed position will remain in place when you scroll down the page. When you position it using the "top," "right," "bottom," and "left" properties, you're telling it, "Hey, position yourself X pixels this way and Y pixels in relation to the browser window. And stay put, no matter how far up or down, right or left, someone scrolls."

This is just an introduction to the differences between the properties. With this bit of knowledge, we can begin exploring more complicated ways to place elements on your page.
