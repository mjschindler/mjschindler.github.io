Ruby is an object oriented coding language. That means that objects are "the things that do stuff" in your Ruby programs. Objects store and share information. Objects perform tasks and calculations. Objects interact and work with each other. But how does an object know what and how to do these things? How does an object know that it is an object?

In Ruby, a class tells an object what to do and how to be. Classes are like a rubber stamp for spitting out many instances of the same class of objects. So you want to make a bunch of kitty_cat objects, but you don't want to type the same code over and over again. What do you do? Just make one stamp of a kitty_cat, and then you can use it STAMP STAMP STAMP out as many as you like. All your kitty_cats will have the same basic characteristics that you told your class -- your stamp -- to have. But you can also differentiate them from each other through unique attributes or variables.

Let's make a kitty_cat class to see this in action. We're going to use the class keyword to create the new class definition:

class KittyCat #In Ruby, class names are capitalized, and names with more than one word should be CamelCase.
attr_reader :name, :breed, :color
def initialize (name, breed, color )
@name=name
@breed = breed
@color = color
end
end
# Creating and using a KittyCat object looks like this:
kitty_cat = KittyCat.new('Macaroon', 'Norwegian Forest Cat', 'Calico')
kitty_cat.name #=> Macaroon
kitty_cat.breed #=> Norwegian Forest Cat
kitty_cat.color #=> Calico
Now you can easily make all the kitty_cats you want!
